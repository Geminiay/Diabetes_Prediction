### Diabetes Screening Pipeline ###

Early detection of Type 2 diabetes is one of the most impactful interventions in preventive medicine. A screening tool that misses a diabetic patient delays treatment and increases the risk of severe complications — including nephropathy, neuropathy, and cardiovascular disease.

This project uses the Pima Indians Diabetes Dataset. It aims to maximize recall value and minimize missed diabetic cases, while maintaining acceptable precision.

-Train/test split performed before any imputation or scaling
-SHAP values integrated into the main pipeline
-Feature engineering thresholds derived from peer-reviewed medical literature

**Result:** Recall improved from 57% (baseline) → 83% (final model), with only 9 missed diabetic cases out of 54 in the test set.

Key Technical Features

1. Data Leakage
The train/test split is the first operation performed on the raw data. All subsequent fitting is done exclusively on the training fold and then applied to the test set.

2. Feature Engineering
The features provided by dataset was not enough so new elements have been created using feature engineering. Some generated features are: Glucose-Insulin ratio, Glucose-BMI interaction and Pregnancies-Age interaction which generated by dataset itself. Other than those, there are 2 features created by defining new ranges. BMI Category defines 6 different classification according to [1]Nuttall, F. Q. (2015) and Age Risk which [2]Beard, J. R. et al. labels each age according to its health conditions.

3. Missing Value Strategy
The dataset contains zero values in Glucose, BloodPressure, SkinThickness, Insulin, and BMI. These are treated as missing, not zero: KNN Imputer (`k=5`) used instead of mean/median.

4. SMOTE
The dataset is imbalanced (~65% negative, ~35% positive). SMOTE generates synthetic diabetic samples. It is applied only to the training set to prevent data contamination.

5. Hyperparameter Tuning
Optuna has been used rather than grid search to find optimal parameters. The best parameters found are,

n_estimators: 438
max_depth: 13
min_samples_split: 4
min_samples_leaf: 1
max_features: sqrt


6. Clinical Threshold Tuning
Default classification threshold (0.5) optimized for accuracy. Threshold reduced to 0.35 using `predict_proba`, shifting the model toward higher sensitivity at the cost of moderate precision loss. This is the standard approach in medical screening tools where false negatives are more costly than false positives.


## Cross-Validation Stability

Fold Scores : [0.578 0.667 0.602 0.639 0.65]
Mean:  0.627
Std:   0.032

A standard deviation of 0.032 indicates strong generalization. The model's performance is consistent across all five folds.

## SHAP Explainability

SHAP values reveal the contribution of each feature to individual predictions. The summary plot consistently shows:

1. **Glucose_BMI** - top predictor, outperforming Glucose alone. High values of both together create a stronger diabetes signal than either individually.
2. **Glucose** - second, behaves as expected. High values push toward diabetic, low values toward healthy.
3. **Insulin** - third, but noisy. Wide spread in both directions, likely reflecting the high missing value rate (48.7%) in the original data.
4. **Age_BMI** - consistent and expected direction. Higher values increase diabetes risk.
5. **DiabetesPedigreeFunction** - moderate but stable contribution. Genetic risk factor working as intended.
6. **Pregnancies_Age** - modest contribution, weaker than Glucose_BMI_Interaction.

This ranking validates that the model is learning clinically meaningful patterns.
